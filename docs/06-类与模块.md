# 06 · 类与模块（Classes & Modules）

> 对应官方：[Classes](https://www.typescriptlang.org/docs/handbook/2/classes.html)、[Modules](https://www.typescriptlang.org/docs/handbook/2/modules.html)

---

## 1. 类与类型

```ts
class Greeter {
  readonly name: string;
  constructor(name: string) {
    this.name = name;
  }
  greet(): string {
    return `Hello, ${this.name}`;
  }
}
```

- `public` / `protected` / `private`（编译期可见性；**不是**运行时强隐私）。
- `readonly`、`abstract` 类与方法。

---

## 2. `implements` 与接口

```ts
interface Drawable {
  draw(): void;
}

class Circle implements Drawable {
  draw() {
    console.log('circle');
  }
}
```

**重点**：`implements` 只检查**实例侧**形状；不检查**静态**部分。

---

## 3. 类表达式与类型侧

类名在类型位置与值位置有不同含义；需要「类的实例类型」时常用 `InstanceType<typeof C>`。

---

## 4. 模块（ES Modules）

- 有 `import` / `export` 的文件为**模块**，顶层作用域为模块作用域。
- **文件即模块**：避免「全局脚本」与模块混用导致难以追踪的副作用。

```ts
// math.ts
export const PI = 3.14;
export function sum(a: number, b: number) {
  return a + b;
}
```

---

## 5. `import type` 与类型仅导入

```ts
import type { SomeType } from './types';
```

利于打包器做摇树优化，并明确「仅类型依赖」。
