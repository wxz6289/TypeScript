# 00 · 定位与目标

> 对应官方：[The TypeScript Handbook — About](https://www.typescriptlang.org/docs/handbook/intro.html)

---

## 1. TypeScript 是什么

- **定位**：JavaScript 的**静态类型检查器**——在代码运行**之前**（静态）检查类型是否合理（typechecked）。
- **产出**：编译（转译）为普通 JavaScript；类型信息**不参与**运行时。
- **常见错误类型**：在需要 `A` 的地方用了 `B`（拼写、API 误用、对运行时行为的错误假设等）。

```ts
// 典型「类型错误」：length 是 number，不能当函数调用
function f(n: number) {
  return n.length(); // 编译期报错
}
```

---

## 2. Handbook 与 Reference 的分工

| 区块 | 作用 | 阅读方式 |
|------|------|----------|
| **Handbook** | 日常写 TS 需要掌握的语法与行为 | 建议按左侧目录顺序通读 |
| **Reference** | 单一主题的深入说明 | 按需查阅，不要求连续阅读 |

Handbook **不是**完整语言规范：为可读性会省略部分边角；正式行为以 Reference 为准。

---

## 3. Handbook 的非目标（官方明确）

- 不系统讲授 JS 的函数、类、闭包等基础（会给出外链建议）。
- 不替代语言规范。
- **基本不**展开与各打包器/框架的深度集成（webpack、Vite、React 等另有专题）。
