# 02 · 日常类型（Everyday Types）

> 对应官方：[Everyday Types](https://www.typescriptlang.org/docs/handbook/2/everyday-types.html)

---

## 1. `object` 与顶层对象类型

在 TS 中，`object` **不是**「任意对象」的宽松含义，而是「非原始类型」：

```ts
function acceptObject(o: object) {}
acceptObject({}); // ok
acceptObject([]); // ok
acceptObject(null); // strictNullChecks 下通常报错
```

需要「任意值」时考虑 `unknown`；需要「键值对象」时用索引签名或具体接口。

---

## 2. 联合类型 `|`

```ts
function printId(id: number | string) {
  console.log('id:', id);
}
```

对联合类型做操作前通常需要**收窄**（见 [03-收窄](./03-收窄.md)）。

---

## 3. 字面量类型

```ts
let direction: 'left' | 'right' = 'left';
direction = 'up'; // 报错
```

与 `const` 推断结合：

```ts
const req = { url: 'https://example.com', method: 'GET' as const };
// method 类型为字面量 "GET"，而非 string
```

---

## 4. 字面量推断与 `as const`

```ts
const cfg = { x: 10, y: 20 }; // x,y 推断为 number
const cfg2 = { x: 10, y: 20 } as const; // 只读 + 字面量类型
```

---

## 5. 可选属性与 `undefined`

```ts
type User = {
  name: string;
  nick?: string; // nick?: string 等价于 nick: string | undefined（在部分 strict 组合下略有差异）
};
```

---

## 6. `readonly`

```ts
type Point = { readonly x: number; readonly y: number };
const p: Point = { x: 1, y: 2 };
p.x = 3; // 报错
```

只读数组：`readonly number[]` 或 `ReadonlyArray<number>`。

---

## 7. 接口 `interface` 与类型别名 `type`

```ts
interface A {
  x: number;
}
interface A {
  y: number; // 声明合并：同一 interface 可多次声明
}

type B = { x: number } & { y: number };
```

**选型提示**：

- 需要**声明合并**、类 `implements` 的公共形状 → 优先 `interface`。
- 联合、交叉、映射、条件类型等**类型运算** → 多用 `type`。
