# 07 · 工程与配置（Project Configuration）

> 对应官方：[What is a tsconfig.json](https://www.typescriptlang.org/docs/handbook/tsconfig-json.html)、[TSConfig Reference](https://www.typescriptlang.org/tsconfig/)

---

## 1. 最小认知

- `tsconfig.json` 定义**编译目标**、**模块系统**、**类型检查严格程度**、**包含哪些文件**。
- `tsc` 按该配置做类型检查与（可选）_emit_。

```bash
npx tsc --init
npx tsc          # 编译
npx tsc --noEmit # 仅类型检查
```

---

## 2. 常用顶层字段

| 字段 | 作用 |
|------|------|
| `compilerOptions` | 编译与类型检查选项 |
| `include` / `exclude` | 包含/排除文件 glob |
| `files` | 显式文件列表 |

---

## 3. `compilerOptions` 重点（必会子集）

| 选项 | 建议 |
|------|------|
| `target` | 输出 JS 语言版本，如 `ES2020` |
| `module` | `NodeNext` / `ESNext` / `CommonJS` 等与运行环境一致 |
| `moduleResolution` | 常与 `module` 联动；Node 现代项目常见 `NodeNext` |
| `strict` | 新项目默认 `true` |
| `skipLibCheck` | 大项目常开以加速；跳过 `.d.ts` 检查 |
| `esModuleInterop` | 改善 CJS 默认导出互操作 |
| `resolveJsonModule` | 允许 `import x from './x.json'` |
| `isolatedModules` | 单文件可安全转译（与 Babel/esbuild 等配合） |
| `noEmit` | 只做检查、不输出（配合单独打包器时常见） |

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "strict": true,
    "skipLibCheck": true,
    "esModuleInterop": true,
    "verbatimModuleSyntax": true
  },
  "include": ["src/**/*"]
}
```

> 具体组合以官方 [TSConfig Reference](https://www.typescriptlang.org/tsconfig/) 为准。
