# 01 · 基础（The Basics）

> 对应官方：[The Basics](https://www.typescriptlang.org/docs/handbook/2/basic-types.html) 及 Handbook 入门章节

---

## 1. 类型注解与类型推断

```ts
// 显式注解
let n: number = 1;

// 推断：右侧是数字字面量 → 推断为 number
let x = 2;
x = 'hi'; // strict 下报错：不能将 string 赋给 number
```

**重点**：开启 `strict` 后，尽量让推断工作；仅在边界（空数组、函数返回值等）需要补注解。

```ts
// 空数组常被推断为 never[]，宜注解元素类型
const ids: number[] = [];
```

---

## 2. `strict` 总开关（必会）

`"strict": true` 会打开一组严格检查，包括但不限于：

- `noImplicitAny`
- `strictNullChecks`
- `strictFunctionTypes`
- `strictBindCallApply`
- …

```ts
function greet(name: string | undefined) {
  console.log(name.toUpperCase()); // strictNullChecks：可能 undefined
}
// 修复：先判断或可选链
function greet2(name: string | undefined) {
  console.log(name?.toUpperCase());
}
```

---

## 3. 原始类型与包装对象（勿混用）

日常只使用 **小写** 原始类型：`string`、`number`、`boolean`、`symbol`、`bigint`。

```ts
// 不推荐：大写 String/Number 等对应包装对象，易与原始类型混淆
const bad: String = 'x';
const good: string = 'x';
```

---

## 4. `any` 与 `unknown`

| 关键字 | 含义 |
|--------|------|
| `any` | 关闭检查；可赋给任意类型，**慎用** |
| `unknown` | 类型安全顶层；使用前必须收窄 |

```ts
function useUnknown(x: unknown) {
  if (typeof x === 'string') {
    console.log(x.toUpperCase());
  }
}
```

## 6. `never` 与穷尽检查

`never` 表示「不可能出现的值」；常用于**穷尽分支**或抛错函数返回值。

```ts
function assertNever(x: never): never {
  throw new Error('unexpected: ' + x);
}

type Shape = { kind: 'circle'; r: number } | { kind: 'square'; s: number };

function area(s: Shape) {
  switch (s.kind) {
    case 'circle':
      return Math.PI * s.r * s.r;
    case 'square':
      return s.s * s.s;
    default:
      return assertNever(s);
  }
}
```

## 7. `null`、`undefined` 与 JSON

- **`null`**：表示「有意的空值」。
- **`undefined`**：表示「未赋值 / 缺失」。

在 `strictNullChecks` 下，二者与具体类型不自动兼容，需显式处理（见上文 `greet` 示例）。

**与 JSON 互操作**：`JSON.stringify` 支持 `null`，但会**忽略**值为 `undefined` 的属性；`JSON.parse` 也不会产生 `undefined` 字段。在 API 边界设计类型时，应区分「字段缺失」与「字段为 null」。
