# 第 3 章 · 类型推断与控制流分析（Type Inference and Control Flow Analysis）

对应原书第 3 章（含 **Item 22: Understand Type Narrowing** 等主题，见原书）。

---

## 1. 本章目标

用**更少注解**获得同等安全：理解 TS 如何在 `if`、`switch`、`return` 等控制流上**缩小**变量的可能类型。

---

## 2. 控制流分析（CFA）直觉

```ts
function fmt(x: string | number) {
  if (typeof x === 'string') {
    return x.toUpperCase();
  }
  return x.toFixed(2);
}
```

- 在 `true` 分支里，`x` 的集合被**剪掉**非 `string` 部分。  
- `typeof`、`in`、`instanceof`、自定义 type predicate 都是「剪刀」。

---

## 3. 判别联合：把「标签」放进类型

```ts
interface User {
  name: string;
}

type Result<T> =
  | { ok: true; value: T }
  | { ok: false; error: string };

function unwrap<T>(r: Result<T>): T {
  if (r.ok) return r.value;
  throw new Error(r.error);
}
```

**要点**：字段 `ok` 把**合法组合**钉死，避免 `value` 与 `error` 同时存在或同时缺失。

---

## 4. 推断 vs 注解：何时动手写类型

| 场景 | 建议 |
|------|------|
| 空数组 `[]` | 注解元素类型，避免 `never[]` |
| 公共 API 导出 | 显式返回类型，防止实现改动「泄漏」到调用方 |
| 复杂字面量 | `as const` + `satisfies`（TS 4.9+）平衡推断与校验 |
| 临时中间变量 | 多依赖推断，减少噪音 |

---

## 5. `??` 与 truthy 收窄的差异

- `x ?? def`：只在 `null` / `undefined` 时回退。  
- `x || def`：对所有 **falsy**（`0`、`''`、`false`）回退——业务里常是 bug 源。
