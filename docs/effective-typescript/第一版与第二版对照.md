## 5. 重大改写条目（第二版 · 作者标注 *Major Changes*）

以下为发布说明中强调的**结论或结构显著变化**（仅列要点，细节以原书为准）。

| 第二版 Item | 相对第一版的要点（归纳） |
|-------------|---------------------------|
| **13** `type` vs `interface` | **结论倾向 `interface`**；补充 `type` 内联等讨论，弱化「带属性的函数类型」示例路径。 |
| **14** `readonly` | 在通向 `ReadonlyArray` 等话题前，强化 **`readonly` 属性修饰符** 的讨论。 |
| **15** 类型运算 / 泛型避免重复 | 原 14 题；**大量示例重写**；弱化「泛型入门」篇幅，加深 **映射类型**（含 homomorphic、`as` 子句等）。 |
| **16** 索引签名 | 原 15 题；**立场反转**：从「用索引签名表达动态数据」转为**优先更精确替代方案**；扩展 `Map`、补充模板字面量类型示例。 |
| **17** 数字索引签名 | 原 16 题；标题改为 **Avoid** Numeric Index Signatures；补充可迭代/生成器相关讨论。 |
| **20** 变量如何获得类型 | 原 21 widening；术语更贴近官方；加入 **`tuple()` 辅助**、**`satisfies`** 等现代写法。 |
| **42** 避免基于轶事数据的类型 | 原 35「从 API/规范生成类型」；更强调**不要凭抽样数据造类型**；示例从 GraphQL 换为 **OpenAPI** 方向。 |
| **45** 隐藏不安全断言 | 原 40；示例由 `cacheLast`/memo 改为 **fetch JSON** 一类更常见场景。 |
| **47** 类型安全的 monkey patch | 原 43；聚焦 **全局对象**（如 `userFromPageLoad`）的建模方式。 |
| **55** 为类型写测试 | 原 52 pitfalls；**几乎重写**；纳入 `expect-type`、类型挑战思路、`eslint-plugin-expect-type` 等。 |
| **70** Mirror types 切断依赖 | 原 51；**论证整体重写**，结论方向仍相关。 |
| **72** 优先 ECMAScript 能力 | 原 53；更新 **decorators** 讨论；并入旧 **Item 56** 关于 `private` 的警示。 |
| **73** Source maps 调试 | 原 57；补充 **Node.js** 调试语境。 |
| **79** 写现代 JavaScript | 原 58；聚焦 **import 与 class**；旧版大段内容压缩为条目列表。 |

> 若你发现「第一版某条在第二版找不到了」，多半已 **合并 / 改名 / 换章**——务必对照原书附录链接。

## 6. 轻微改动条目（*Minor Changes*）

发布说明列出大量 **Item 编号仍保留或仅微调标题** 的条目（例如 Item 1–12、18–27、29–35、40–41、43–44、46、49、52、60–69、75、80–83 等），总体为：

- 措辞与 **eslint 规则引用** 更新；  
- 示例与 **TS 5** 语法对齐；  
- 个别标题从「Recognize Limits…」改为更直接的 **Distinguish…** 等。

**第一版读者**：若已内化旧版结构，第二版不必重读每一字，但建议至少过一遍 **§4 全新 Item + §5 重大改写** 对应章节。

## 8. 自检：你是否补齐了「第一版没强调」的能力

- [ ] 能解释 **条件类型在联合上的分发** 何时需要关闭。  
- [ ] 能为 API 写 **模板字面量类型** 约束字符串关系（而不仅是 `string`）。  
- [ ] 知道何时用 **代码生成** 替代「过深」的类型体操。  
- [ ] 会用 **`never` 穷尽** 把遗漏分支变成编译错误。  
- [ ] 理解 **类型检查与单元测试** 的分工与重叠。  
- [ ] 关注 **`tsc`/IDE** 在大仓库中的性能策略。
