# 第 6 章 · 泛型与类型层编程（Generics and Type-Level Programming）

对应原书第 6 章（第二版新增大块内容；公开资料提及类型层可表达复杂计算、**Item 55: Write Tests for Your Types** 等）。

---

## 1. 本章定位

把泛型从「容器里装任意 T」提升到：**在类型层表达约束、映射与推导**，减少重复与运行时分支。

---

## 2. 泛型约束：`extends` 表达「最小能力」

```ts
function pick<T extends object, K extends keyof T>(obj: T, keys: K[]): Pick<T, K> {
  const out = {} as Pick<T, K>;
  for (const k of keys) out[k] = obj[k];
  return out;
}
```

---

## 3. 条件类型与 `infer`

```ts
type ElementType<T> = T extends (infer U)[] ? U : T extends readonly (infer U)[] ? U : never;

type N = ElementType<number[]>; // number
```

**注意**：分布式条件类型在联合上「分发」，组合复杂时需刻意用 `[T] extends [...]` 关闭分发（书中会讲模式）。

---

## 4. 映射类型：批量增删改属性修饰符

```ts
type Mutable<T> = { -readonly [K in keyof T]: T[K] };
```

---

## 5. 模板字面量类型

```ts
type HttpMethod = 'GET' | 'POST';
type Path = '/users' | '/posts';
type Route = `${HttpMethod} ${Path}`;
```

---

## 6. 「为类型写测试」

- 对复杂条件类型，用 `type Expect<T extends true> = T` + `===` 断言期望相等。  
- 把易回归的类型行为锁进 CI，而不是仅靠示例代码。

```ts
type Eq<A, B> = (<T>() => T extends A ? 1 : 2) extends <T>() => T extends B ? 1 : 2
  ? true
  : false;
type _t = Eq<ElementType<string[]>, string>;
```
