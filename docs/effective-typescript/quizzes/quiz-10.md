# 第 10 章自测

**Q1** 迁移大项目时，「按目录开 strict」与「一次性 strict」如何权衡？  
**思路**：渐进路径、边界目录、技术债登记。

**Q2** 如何用类型覆盖率指标驱动迁移优先级？  
**思路**：热点目录、`any` 密度、业务关键路径。

**Q3** 解释「类型即文档」在 API 评审中的用法。  
**思路**：签名即契约；breaking change 从类型 diff 可见。

**Q4** 团队约定「禁止双断言」的理由？  
**思路**：soundness 洞；用校验或单点封装替代。

**Q5** 学完本书后，你个人「三条铁律」会写什么？（开放题）  
**思路**：自答：如 unknown 边界、判别联合、少 any。
