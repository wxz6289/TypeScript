# 第 1 章自测

**Q1** 为什么说「通过 `tsc`」不等于「运行时一定不会类型相关崩溃」？  
**思路**：类型擦除；编译期错误已拦下，但逻辑错误、外部数据未校验仍可能崩。

**Q2** 结构类型下，为何「类名不同」的两个对象可能互相赋值？  
**思路**：比较形状；与名义类型语言对比。

**Q3** 举一例说明 `any` 如何破坏「跳转到定义」与重构安全。  
**思路**：`any` 上属性访问无检查；改名不报错。

**Q4** `strictNullChecks` 打开后，哪些常见代码会最先变红？  
**思路**：未处理 `null`/`undefined` 的解引用；可选链与判空。

**Q5** `noEmit: true` 的典型使用场景？  
**思路**：CI 仅类型检查；实际 bundle 交给 esbuild/webpack。
