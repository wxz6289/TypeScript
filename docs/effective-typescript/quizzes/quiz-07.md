# 第 7 章自测

**Q1** 何时 `Promise.all` 的返回类型会「变窄」或「变宽」？  
**思路**：同质 `T[]` vs 异质元组；失败短路行为。

**Q2** 如何把「可能抛」的函数包装成 `Result` 型而不丢类型信息？  
**思路**：泛型 `tryCatch` 返回 `E extends Error ? ...`；或 `never` 在联合中表达。

**Q3** `async` 函数显式标注返回 `Promise<User>` 与让推断各有什么坑？  
**思路**：标注过宽丢联合收窄；推断可能把错误类型吞进 `any`。

**Q4** 解释「异步错误」在类型层面的表达难点。  
**思路**：`throw` 无类型；用 `Result` 或 `never` 联合显式化。

**Q5** 列举一种避免「双重 await」误用的类型技巧。  
**思路**：区分 `T` 与 `Promise<T>` 的重载或条件返回。
