# 第 10 章 · 现代化与迁移（Modernization and Migration）

对应原书第 10 章（公开摘要提及迁移前清理死代码、**渐进迁移**、度量进展与业务收益等）。

---

## 1. 迁移前准备

- **删除死代码**、去掉已废弃 API：减少需要标注与推理的表面积。  
- 大型 UI 框架切换（如 jQuery → React）通常**晚于**先上 TS：类型会显著降低重构风险。

---

## 2. 渐进策略

- 单目录/单包先行 `allowJs` + `checkJs` 或 `.ts` 旁路迁移。  
- 用 `// @ts-nocheck` **仅作临时舱门**，配清单与移除计划。  
- 跟踪「`any` 数量、错误数、迁移覆盖率」等指标，避免回潮。

---

## 3. 业务理由（为何值得做）

原书引用业界数据说明「可预防的 bug 比例」——具体数字以原书为准；工程上应建立**你们团队自己的**前后对比（缺陷率、返工时间、上手时间）。

---

## 4. 与工具链协同

- ESLint：`typescript-eslint` 规则与 `tsc` 分工：前者偏风格与部分语义，后者是类型真相源之一。  
- CI：`tsc --noEmit` 与测试并行门禁。
