# 附录 · 核心实践清单（原创归纳）

下列清单与**原书 83 条 Item** 主题对应，**不做逐条抄录**；用于自测与 Code Review。细颗粒度表述请以原书为准。**第一版读者**请同时阅读 [第一版与第二版对照](./第一版与第二版对照.md)。

---

## A. 语言与心智模型（对应第 1–2 章）

- [ ] 能解释：类型擦除、编译期错误 vs 运行时异常。  
- [ ] 能解释：结构类型与「同名类型」无关。  
- [ ] 明确团队 `tsconfig`（至少 `strict`、模块策略、`moduleResolution`）。  
- [ ] 把类型当作**值的集合**思考赋值与联合/交叉。  
- [ ] 分清类型空间与值空间；避免无意同名。  
- [ ] 优先注解；断言是最后手段且应局部化。  
- [ ] 用类型级工具消除重复字段声明。

---

## B. 推断、收窄与设计（对应第 3–4 章）

- [ ] 能不用断言写出判别联合与穷尽 `switch`。  
- [ ] 不为「不可能组合」留布尔旗标堆。  
- [ ] 谨慎使用 `||` 与 `??`，明确 falsy 业务语义。  
- [ ] 公共 API 对「空数组、宽返回」显式标注。  
- [ ] 用 `satisfies` / `as const` 等在推断与校验间取得平衡（版本允许时）。

---

## C. `any`、不健全与逃生舱（对应第 5 章）

- [ ] 统计并下降 `any`；边界用 `unknown`。  
- [ ] `@ts-expect-error` 优于长期 `@ts-ignore`。  
- [ ] 禁止「双 unknown 断言」成为常态。  
- [ ] 知道团队代码里**已知的 unsound 角落**（第三方声明、初始化豁免等）。

---

## D. 泛型与类型层（对应第 6 章）

- [ ] 泛型参数有清晰上界 `extends`。  
- [ ] 条件类型分发行为可预测；必要时用元组包装关闭分发。  
- [ ] 复杂类型有条件维护「类型测试」。  
- [ ] 模板字面量类型用于字符串枚举 API 时有上限（可读性）。

---

## E. 配方思维（对应第 7 章）

- [ ] 对「遗漏分支」使用 `never` 穷尽。  
- [ ] 对「过滤 nullish」使用可复用谓词收窄。  
- [ ] 不为炫技引入团队无法维护的类型体操。

---

## F. 声明文件与生态（对应第 8 章）

- [ ] `typescript` 与 `@types` 依赖分区正确。  
- [ ] 优先使用维护良好的 `@types` 而非巨型本地 shim。  
- [ ] `.d.ts` 与 TSDoc 不重复表达类型已写明的事实。

---

## G. 运行与工程（对应第 9 章）

- [ ] 新功能优先标准 ES 语法。  
- [ ] 外部输入必有运行时校验层。  
- [ ] 关注 `tsc`/IDE 性能与大仓库拆分策略。

---

## H. 迁移（对应第 10 章）

- [ ] 迁移路线可量化、可回滚、可跟踪。  
- [ ] 先减死代码再迁类型。  
- [ ] CI 门禁：`tsc --noEmit` + 测试。
